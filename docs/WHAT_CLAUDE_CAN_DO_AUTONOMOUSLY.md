# What Claude Can Do Autonomously (and what it will always ask you first)

Full policy: `system/AUTONOMY_POLICY.md`. Simple version below.

## Without asking (reversible, in-scope, audited by Git)
- Create, triage, research, and move queue tasks; run and reschedule watches.
- Capture timeline observations, file research with sources, link and supersede artifacts.
- Build, edit, and publish the newspaper (through the Publisher gate).
- Process your annotations into tasks/evidence/watches/corrections/proposals.
- Record preference EVIDENCE and write PROPOSED rules.
- Create domains, rebuild indexes/dashboards, run tests, repair generated files.
- Commit and push within the granted repos, with push verification.
- Run background research within capacity policy (deadline work first).
- Spawn cheaper/stronger models per routing policy; add reviewers for consequential work.

## With an audit note (does it, but leaves a visible record)
Reorganizing folders · merging duplicates (originals kept) · changing staffing · schema
version bumps · adjusting one edition's budgets · syncing skill updates across repos
(operation-ledger tracked).

## NEVER without your explicit confirmation
- Confirming a rule into `preferences/CONFIRMED_RULES.md` from inferred evidence.
- Deleting real knowledge or any sensitive history (forgetting stops at the PLAN).
- Rewriting Git history anywhere.
- Editing a robot's operating prompt (proposes a reviewable diff instead — the 2026-07-10
  application happened because you explicitly authorized it in writing).
- Anything external: sending email (drafts only, ever), applying to jobs, real trades or
  brokerage connections, purchases, new accounts/credentials/permissions.
- Moving anything across the personal/work boundary, in either direction.
- Structural changes to specialist repos beyond the integration surface.

## How you change these boundaries
Edit `system/AUTONOMY_POLICY.md` (it's yours), or say the change in a session —
it becomes a reviewed edit with a Git trail, like everything else.
