# Retrieval Policy

Retrieval must find relevant memory without loading the whole Brain, and must not surface
sensitive artifacts out of context.

## Mechanism (v1 — deliberately simple)
1. `python3 tools/build_index.py` → `system/INDEX.tsv`: one row per artifact
   (id, path, artifact_type, domain, status, sensitivity, topics, entities, title,
   created_at). Rebuildable from Markdown; never authoritative; regenerated by tools that
   write artifacts and by any session that finds it stale.
2. `python3 tools/brain_search.py "<query>" [--domain d] [--type t] [--since date]
   [--allow-sensitive REASON]` — ranks by: term hits in title/topics/entities (weighted),
   term hits in body (grep), domain match, recency. Sensitivity gate per PRIVACY_POLICY:
   health/private/financial rows are filtered out unless --domain matches the artifact's
   domain or --allow-sensitive is given (reason required, logged to stderr for the caller
   to record).
3. For a task, retrieve in this order: same-domain knowledge + preferences → open questions
   → related timeline (entity/topic match) → predictions/outcomes on the same entities →
   cross-domain only via non-sensitive artifacts.

A SQLite/vector layer is NOT included: with <1000 files, grep+index passes the retrieval
tests (tests/test_retrieval.sh). Revisit when the index exceeds ~5k rows or tests show
recall failures — record the evidence first (nonnegotiable principle 6).

## Required behaviors (tested)
- Later symptom retrieves earlier related timeline entry within the health domain, WITHOUT
  causal claims (the artifact text carries the uncertainty).
- Vehicle query retrieves prior cab/engine/mileage/location preferences.
- Fantasy query retrieves league scoring facts and Brendan's prior player views.
- Stock query retrieves the original thesis unmodified (supersede chains intact).
- Unrelated query does NOT retrieve sensitive health artifacts.
