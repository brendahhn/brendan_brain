# Skill Registry — interface version 1.0.0

Canonical source for ALL Brendan OS skills: `brendan_brain/.claude/skills/`. Synced copies
in specialist repos are generated by `tools/sync_skills.sh` (checksums in
`skills/sync_manifest.tsv`; verify with `tools/sync_skills.sh --check`). Never edit a
synced copy — the header comment in each SKILL.md says where the canonical lives.

## Distribution strategy & platform limitations (documented honestly)
- **Cloud routines**: skills load from the repo's `.claude/skills/` at session start —
  synced copies make them available inside each robot's scheduled runs. The robot must have
  `brendan_brain` in its routine repo selection for `../brendan_brain` to exist.
- **Local use**: Brendan can symlink or copy `brendan_brain/.claude/skills/*` into
  `~/.claude/skills/` for user-scoped availability; documented in docs/OPERATIONS.md.
- **Limitation**: there is no platform mechanism to push a skill update into already-cloned
  repos automatically; sync happens when someone runs `tools/sync_skills.sh` in a session
  that has the repos side by side (like this one). Drift is detectable via `--check` and
  the version headers; brain-sync also self-checks its major version against this file at
  robot runtime.
- Interface version bumps: minor = compatible additions; major = format changes — update
  this file, the skill headers, and re-sync everywhere in one operation (tools/oplog.py).
